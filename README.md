## Project Overview

This project investigates whether the quality of NLP signals generated by different large language models (LLMs) affects the trading performance of a deep reinforcement learning (DRL) agent. We compare four LLM configurations — Gemini 3.1 Pro, Qwen3.5-27B (base), Qwen3.5-27B with QLoRA, and Qwen3.5-27B with QA-LoRA — within the PrimoGPT/PrimoRL framework adapted from Botunac (2025).

**Key Finding:** LLM choice has no significant effect on DRL trading performance (ANOVA p=0.998, η²=0.006). However, an ablation study reveals that NLP-only agents outperform Tech-only agents (+1.8pp, p<0.0001), which we attribute to information redundancy and simpler state space design rather than signal quality.

## How to Reproduce

### 1. LLM Fine-Tuning & Inference (Google Colab)

Open `code/SFT/llm_finetune_inference.ipynb` in Google Colab with a GPU runtime. The notebook supports 3 configurations:
- **Vanilla Qwen** (no fine-tuning, inference only)
- **QLoRA** fine-tuning on PrimoGPT training data
- **QA-LoRA** fine-tuning on PrimoGPT training data

Upload `merged_data.json` (training data) and stock news CSVs to your Google Drive `primogpt_data/` folder as instructed in the notebook.

Pre-generated NLP signals and fine-tuned model weights are available in the [Google Drive folder](https://drive.google.com/drive/folders/1ILoTTr4r08_hejFeVCj8HBfzwwITbcsy).

### 2. NLP Signal Generation

```bash
# Generate signals for each LLM configuration
python code/nlp/generate_signals.py --config gemini
python code/nlp/generate_signals.py --config qwen_base
python code/nlp/generate_signals.py --config qwen_qlora
python code/nlp/generate_signals.py --config qwen_qalora
```

### 3. PPO Training (v7 Definitive Experiment)

```bash
# 30-seed experiment with all 5 LLM configs
python code/training/run_deep_30seed.py
```

### 4. Ablation Study (v8)

```bash
# Full / Tech-only / NLP-only feature sets
python code/training/run_ablation_parallel.py
python code/training/run_ablation_nlp_only.py
```

### 5. Analysis

```bash
# Statistical analysis
python code/analysis/analyze_ablation.py
python code/analysis/compare_returns.py
```

## Contributions Relative to Original Codebase

This project builds upon the open-source PrimoGPT/PrimoRL codebase by Botunac (2025). Our novel contributions include:

1. **Multi-LLM comparison framework** with 4 LLM configurations (original used only QA-LoRA)
2. **LLM fine-tuning pipeline** — QLoRA and QA-LoRA fine-tuning via Colab notebook with inference
3. **Look-ahead bias fix** — NLP shift(1) to prevent future information leakage
4. **Transaction cost integration** — 0.1% per trade (original had none)
5. **30-seed experimental design** with ANOVA statistical framework (original: single seed)
6. **Ablation study** — Full / Tech-only / NLP-only feature set comparison
7. **Feature importance analysis** via permutation importance
8. **Ensemble aggregation** and individual-level analysis methodology
9. **Iterative methodology refinement** across 8 experimental versions
