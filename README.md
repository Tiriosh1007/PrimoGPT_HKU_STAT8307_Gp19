# STAT8307 NLP Course Project - Group 19

**Does Better NLP Signal Quality Improve DRL Trading Performance?**

*Mak Chi Lam | Student ID: 3035279753*

## Project Overview

This project investigates whether the quality of NLP signals generated by different large language models (LLMs) affects the trading performance of a deep reinforcement learning (DRL) agent. We compare four LLM configurations — Gemini 3.1 Pro, Qwen3.5-27B (base), Qwen3.5-27B with QLoRA, and Qwen3.5-27B with QA-LoRA — within the PrimoGPT/PrimoRL framework adapted from Botunac (2025).

**Key Finding:** LLM choice has no significant effect on DRL trading performance (ANOVA p=0.998, n=0.006). However, an ablation study reveals that NLP-only agents outperform Tech-only agents (+1.8pp, p<0.0001), which we attribute to information redundancy and simpler state space design rather than signal quality.

## How to Reproduce

### 1. NLP Signal Generation

```bash
# Generate signals for each LLM configuration
python code/nlp/generate_signals.py --config gemini
python code/nlp/generate_signals.py --config qwen_base
python code/nlp/generate_signals.py --config qwen_qlora
python code/nlp/generate_signals.py --config qwen_qalora
```

### 2. PPO Training (v7 Definitive Experiment)

```bash
# 30-seed experiment with all 5 LLM configs
python code/training/run_deep_30seed.py
```

### 3. Ablation Study (v8)

```bash
# Full / Tech-only / NLP-only feature sets
python code/training/run_ablation_parallel.py
python code/training/run_ablation_nlp_only.py
```

### 4. Analysis

```bash
# Statistical analysis
python code/analysis/analyze_ablation.py
python code/analysis/compare_returns.py
```

### 5. Report Compilation

```bash
cd report/
# Requires tectonic or pdflatex with natbib
tectonic main.tex
```

### 6. Presentation Generation

```bash
cd presentation/
npm install
node generate_pptx.js
```

## Contributions Relative to Original Codebase

This project builds upon the open-source PrimoGPT/PrimoRL codebase by Botunac (2025). Our novel contributions include:

1. **Multi-LLM comparison framework** with 4 LLM configurations (original used only QA-LoRA)
2. **Look-ahead bias fix** — NLP shift(1) to prevent future information leakage
3. **Transaction cost integration** — 0.1% per trade (original had none)
4. **30-seed experimental design** with ANOVA statistical framework (original: single seed)
5. **Ablation study** — Full / Tech-only / NLP-only feature set comparison
6. **Feature importance analysis** via permutation importance
7. **Ensemble aggregation** and individual-level analysis methodology
8. **Iterative methodology refinement** across 8 experimental versions
